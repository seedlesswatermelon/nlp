Introduction to Natural Language Processing，自然语言处理介绍¶

NLP是数据科学的一个分支，包括智能和高效地从文本数据中分析、理解和导出信息的系统流程。通过NLP及其组成部分，企业可以组织海量文本数据块、执行许多自动化任务、并解决广泛问题，如自动摘要、机器翻译、命名实体识别、关系抽取、情感分析、语音识别、主题分割等。

Text Preprocessing，文本预处理¶

  • Noise Removal，噪声去除

    问题说明，与数据上下文和最终输出无关的文本片段（可统一称作 噪声）

    主要包括，语言停用词（常用词 is am the of等）、标点符号、URL、标签等

    通常做法，准备 噪声实体字典，通过标记符号来迭代文本对象；或者，使用正则表达式来进行替换

# Sample code to remove noisy words from a text
noise_list = ["is", "a", "this", "..."]

def _remove_noise(input_text):

    words = input_text.split()
    
    noise_free_words = [word for word in words if word not in noise_list]
    noise_free_text = " ".join(noise_free_words)

    return noise_free_text

_remove_noise("this is a sample text")

# Sample code to remove a regex pattern
import re
regex_pattern = "#[A-Za-z0-9\w]*"

def _remove_regex(input_text, regex_pattern):

    urls = re.finditer(regex_pattern, input_text)

    for i in urls:

        input_text = re.sub(i.group().strip(), '', input_text)

    return input_text

  • Lexicon Normalization，词汇标准化

    问题说明，词语（尤其是英文单词）表现的多重表征，如何把一个词的所有差异转换成它们的标准化形式

    主要包括，后缀（ing、ly、es、s、er）、词根（语料学习获得）

    通常做法，stemming，词干提取，基于规则的从一个词剥离后缀的过程（ing、es..）;lemmatization，词变体归类，有组织且有步骤获取词的词根形式的过程，即词汇用法（单词的词典重要性）和形态逻辑分析（词汇结构和语法关系）

  • Object Standardization，对象标准化

    问题说明，任何标准语义字典中不存在的单词或短语（可统一称作 碎片）

    主要包括，缩写词，口语俚语等

    通常做法，借助正则表达式和手工编写的数据词典

In [19]:
lookup_dict = {'rt':'Retweet', 'dm':'direct message', "awsm" : "awesome", "luv" :"love"}

def _lookup_words(input_text):

    words = input_text.split()

    new_words = []

    for word in words:

        if word.lower() in lookup_dict:

            word = lookup_dict[word.lower()]

        new_words.append(word) 
    
    new_text = " ".join(new_words)
    return new_text

_lookup_words("RT this is a retweeted tweet by Shivam Bansal")
Out[19]:
'Retweet this is a retweeted tweet by Shivam Bansal'

Text to Features，文本到特征¶

  • Syntactic Parsing，句法解析

句法解析涉及句中单词的语法和显示这些单词间关系的排列方法的分析，依存语法和部分语音标签是文本句法的重要属性。

Dependency Parsing，DP，依存语法，通过分析语言单位内成分之间的依存关系揭示其句法结构。 直观来讲，依存句法分析识别句子中的“主谓宾”、“定状补”这些语法成分，并分析各成分之间的关系。

  • Entity Extraction，实体提取

实体，句子中最重要的句块 名次短语、动词短语等。

实体检测算法，通常是基于规则解析、字典查找、POS标签、依存句法分析的集成模型，常用的实体检测算法主要是 主题xx模型和命名实体识别。

Named Entity Recognition，NER，命名实体识别，检测如人名、地名、公司名等命名实体的过程。

典型NER模型一般包括：

* 名词短语识别，涉及使用依存解析和词性标注从文本中提取所有名次短语
* 短语分类，将所有被提取名词短语划分为所属相应类别的分类步骤（谷歌地图API提供了消除歧义位置；DBpedia、wikipedia开放数据库可用来识别个人姓名或公司名称；结合不同信息员的查找表和词典进行精确查找）
* 实体消歧，有时可能出现误判分类，通常在分类结果上增加验证层（可以的方案是 流行的知识图 谷歌、IBM沃森、维基百科等）

Topic Modeling，主题模型 是一个存在于文本语料库中主题的自动识别过程，它以无监督方式推导出语料库中的隐含模式，通常使用 LDA，隐含狄利克雷分配

N-Grams as Features，N元连续模型

N-gram，N个单词（N>1)在一起的组合，相比单词（1gram）包含更多信息，且双单词（n=2）被认为是最重要的特征。

In [35]:
def generate_ngrams(text, n):

    words = text.split()

    output = [] 

    for i in range(len(words)-n+1):

        output.append(words[i:i+n])

    return output

generate_ngrams('this is a sample text', 2)

  • Statistical Features，统计特征

文本数据量化为数字

Term Frequency-Inverse Document Frequency，词频-逆文档频率

TF-IDF 是不考虑精确排序的，在文档中出现词汇的基础上，将文本文件转化为向量模型，是常见的信息检索问题加权模型。

TF 的T是文档D中词项T的计数。 IDF 是T在语料库中文档数与包含词项T的文档数的比值的对数，详见 TF-IDF相关描述

count/Density/Readability Features，计数/密度/可读性特征

这些功能似乎微不足道，但在学习模型显示了很大影响。一些特征值有：单词数、句子计数、标点符号数和特定行业用词数

  • word Embedding，text vectors，字嵌入，词向量

字嵌入的目的是通过保留语料库中的上下文相似度，来重新对高维字特征定义到低维特征向量，适用于深度学习模型，常用模型有Word2Vec和GloVe。

Worf2Vec模型是由Continuous Bag of Words和skip-gram两个浅层神经网络模型组成。首先构建一个词汇训练语料库，然后学习字嵌入表征值。它们可以用来作为ML模型的特征向量，使用余弦相似性技术、词聚类和文本分类技术来测量文本相似性。
